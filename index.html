Help Section

Welcome to Maniac OS. This is an amazing Operating System which is capable of handling more than one scheduling algorithm. Moreover, you can even change the size of lists, probability to create a new process, and more! In order for you to understand it a little bit better, you have this helpful section.

Parameters values

  • Probability: [0, 100]
  • Probability to change Page of a single Process: [0, 100]
  • Probability of a process to need IO: [0,100]
  • Quantum: [2, 100]
  • Average CPU (nedded): [1, 100]
  • Average IO (nedded):[1,100]
  • Average Disk (nedded): [1,100]
  • Frame Size: [The amount given]
  • Ram Size: [The amount given] It changes according to the Frame Size.
  • Hold, ready, waiting, waiting for disk: [1, 50]
  • Speediness: Slow (2s), Normal (1.5s) y Fast (1/10s)
  • Algorithm: First Come, First Served and Round-Robin
  • Paging Algorithm: Oldest, least used

Rules

  • First of all you need to select a speed and algorithm from the parameters list.
  • The buttons (play, pause, stop) are in the bar on the top, with Clock timer and Quantum value as well.
  • To make a succesful change of any parameter, first you need to stop or pause the simulation. Then click Play again.
  • Almost any parameter can be modified, except one: the probability of a process to require IO or not. It is all the time 50/50.

PCB Elements

  • ID: The corresponding Process' ID.
  • Arrival: The clock tick the process was done.
  • CPU Nedded: The amount of time it needs in Running.
  • CPU Ussage: The amount of process it has accumulated.
  • Size: The amount of Kb's the whole process needs.
  • Frames: Amount of frame it needs according to the paramaters.
  • Printer Time Nedded: Time it needs to be processed by Printer
  • Time of Printer: Time in printer it has accumulated.
  • Arrival to Printer: Clock tick of the process it should arrived to Printer
  • Waiting Time: Time waiting....
  • System Time: Total time the process was in System
  • Finish Time: Last clock tick in System
  • Efficiency: WaitingTime/TotalTime
  • Status: [In System, Finished, Aborted]

TAP Structure

  • The tap number + L means the location (RAM, Disk or Not Loaded)
  • U means the amount of cpu time it has accumulated.
  • T means the time that page was loaded to Ram, it will turn 0 again if it goes to Disk.
